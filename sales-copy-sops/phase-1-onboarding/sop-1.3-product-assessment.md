# SOP 1.3: Product Deep-Dive Assessment

## Purpose
Conduct a thorough assessment of the product to understand its features, benefits, unique aspects, and potential angles before the kick-off call. This forms the foundation for intelligent questioning and strategic copy direction.

---

## Inputs Required

| Input | Source | Required |
|-------|--------|----------|
| Product access or detailed documentation | Task 1.1 | Yes |
| Classification report | Task 1.2 | Yes |
| Any existing marketing materials | Task 1.1 | Optional |

---

## Step-by-Step Procedure

### Step 1: Product Experience Session
**[TIME: 30-120 minutes depending on product type]**

Actually experience the product. This varies by product type:

#### For Digital Courses:
- [ ] Go through onboarding process
- [ ] Watch/read at least 20-30% of content (first module + sampling)
- [ ] Note the teaching style and methodology
- [ ] Review any workbooks, templates, or resources
- [ ] Check community/support elements if included
- [ ] Note technical quality (video, audio, platform)

#### For SaaS Products:
- [ ] Complete onboarding flow
- [ ] Use core features for actual tasks
- [ ] Explore secondary features
- [ ] Test integrations if applicable
- [ ] Note UX friction points
- [ ] Review help documentation

#### For Supplements:
- [ ] Review all label information thoroughly
- [ ] Research each ingredient individually
- [ ] Note dosages vs. clinically effective dosages
- [ ] Understand the formulation logic
- [ ] Check for unique delivery mechanisms
- [ ] Review manufacturing/quality claims

#### For Coaching/Services:
- [ ] Review service description thoroughly
- [ ] Understand the process/methodology
- [ ] Review any frameworks or systems used
- [ ] Look at deliverables clients receive
- [ ] Understand the transformation journey
- [ ] Review coach/provider credentials deeply

### Step 2: Features Extraction
**[TIME: 20-30 minutes]**

Create a comprehensive features list using this structure:

```markdown
## Features Inventory

### Core Features (What they get)

| Feature | Description | Category |
|---------|-------------|----------|
| [Feature 1] | [What it is] | [Core/Bonus/Support] |
| [Feature 2] | [What it is] | [Core/Bonus/Support] |

### Detailed Feature Breakdown

#### Feature: [Name]
- **What it is:** [Description]
- **How it works:** [Mechanism]
- **Why it matters:** [Relevance to customer]
- **Unique aspect:** [What makes it different]
- **Proof available:** [Any evidence of effectiveness]
```

### Step 3: Benefits Translation
**[TIME: 20-30 minutes]**

For each feature, identify the benefit chain:

```
Feature → Advantage → Benefit → Emotional Payoff

Example:
Feature: 24/7 AI chat support
Advantage: Get answers anytime without waiting
Benefit: Never get stuck; continuous progress
Emotional Payoff: Confidence that you won't fail; peace of mind
```

Complete this for top 10-15 features:

```markdown
## Benefits Translation Matrix

| Feature | Advantage | Benefit | Emotional Payoff |
|---------|-----------|---------|------------------|
| [Feature 1] | [What it enables] | [Result they get] | [How they feel] |
| [Feature 2] | [What it enables] | [Result they get] | [How they feel] |
```

### Step 4: Unique Mechanism Identification
**[TIME: 20-30 minutes]**

This is critical. Identify what makes this product WORK differently:

**Questions to answer:**

1. **What's the core methodology/approach?**
   - Is there a named system or framework?
   - What's the philosophy behind it?
   - What makes the approach different?

2. **What's the "secret sauce"?**
   - Unique ingredient/component?
   - Proprietary process?
   - Novel combination?
   - Counter-intuitive approach?

3. **Why do other solutions fail?**
   - What do competitors miss?
   - What's the common mistake in this space?
   - What's the hidden obstacle most solutions don't address?

4. **Why does THIS solution work?**
   - What does it do that others don't?
   - What's the breakthrough insight?
   - What makes it inevitable vs. hopeful?

Document your findings:

```markdown
## Unique Mechanism Analysis

### The Problem with Current Solutions
[What most solutions in this space get wrong]

### The Hidden Obstacle
[The real reason people struggle - that others don't address]

### The Breakthrough Insight
[What this product understands that others don't]

### The Unique Mechanism
**Name:** [Give it a catchy name if one doesn't exist]
**What it is:** [Brief description]
**Why it works:** [The logic/science behind it]
**How it's different:** [Contrast with alternatives]

### Supporting Evidence
- [Evidence point 1]
- [Evidence point 2]
- [Evidence point 3]
```

## AI Agent Instructions

### Agent: Product Analyst

**Trigger:** Product classification complete, materials available

**Instructions:**
```
You are a Product Analysis Agent. Your job is to deeply analyze products and extract
copy-relevant insights.

When given product information (documentation, features, materials):

1. EXTRACT all features and categorize them:
   - Core features (main value)
   - Bonus features (additional value)
   - Support features (help/access)
   - Technical features (platform/delivery)

2. TRANSLATE features to benefits using the chain:
   Feature → Advantage → Benefit → Emotional Payoff

3. IDENTIFY the Unique Mechanism:
   - What does this product do differently?
   - Why do other solutions fail?
   - What's the breakthrough insight?
   - Give it a catchy name if none exists

4. CATALOG all proof elements:
   - Testimonials (rate strength 1-5)
   - Data/statistics
   - Credentials
   - Scientific support

5. ANALYZE competitive positioning:
   - Who are the alternatives?
   - What are their weaknesses?
   - How is this product differentiated?

6. IDENTIFY story opportunities:
   - Origin story potential
   - Customer transformation patterns
   - Interesting facts/anecdotes
   - Useful analogies

7. FLAG gaps:
   - Missing proof
   - Unanswered questions
   - Potential objections
   - Bonus opportunities

8. GENERATE the complete Product Assessment Document

OUTPUT FORMAT:
Complete Product Assessment Document (markdown)

CRITICAL REQUIREMENTS:
- Be specific, not generic
- Identify UNIQUE aspects, not obvious ones
- Focus on copy-relevant insights
- Flag anything that needs clarification in kick-off call
```

### Agent: Unique Mechanism Identifier

**Trigger:** Product analysis in progress

**Instructions:**
```
You are a Unique Mechanism Identification Agent. Your job is to find the
"secret sauce" that makes a product work differently.

For any product, answer these questions:

1. THE PROBLEM QUESTION:
   "Why do most people fail to achieve [desired result] despite trying?"
   - What's the REAL obstacle (not the obvious one)?
   - What do other solutions miss?

2. THE INSIGHT QUESTION:
   "What does this product understand that others don't?"
   - What's the counter-intuitive truth?
   - What's the hidden leverage point?

3. THE MECHANISM QUESTION:
   "What specifically does this product do that creates the result?"
   - Is it a unique ingredient/component?
   - Is it a proprietary process?
   - Is it a novel combination?
   - Is it a different sequence/timing?
   - Is it addressing a different root cause?

4. THE PROOF QUESTION:
   "What evidence supports that this mechanism works?"
   - Studies
   - Case studies
   - Logical arguments
   - Expert endorsements

OUTPUT FORMAT:
## Unique Mechanism Analysis

### Why Other Solutions Fail
[Specific reason - not "they don't work"]

### The Hidden Truth
[Counter-intuitive insight]

### The Unique Mechanism
**Name:** [Catchy, memorable name]
**What It Is:** [Clear description]
**How It Works:** [Step-by-step if applicable]
**Why It Works:** [Logic/science]
**Proof Points:** [Supporting evidence]

### How to Present This in Copy
[Specific recommendations for communicating this]

IF NO CLEAR MECHANISM EXISTS:
- Flag this as a gap
- Suggest potential mechanisms to explore with client
- Recommend research directions
```

## Common Issues & Solutions

| Issue | Solution |
|-------|----------|
| No product access provided | Work with documentation; note limitations; request access |
| Product is too complex to fully review | Sample strategically; focus on core features |
| Can't identify unique mechanism | Note as gap; prepare specific questions for kick-off |
| Product seems generic/commoditized | Look harder for unique angles; focus on founder story |
| Testimonials are weak or generic | Flag as gap; identify specific proof needs |
