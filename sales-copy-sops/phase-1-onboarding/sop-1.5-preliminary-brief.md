# SOP 1.5: Preliminary Brief Creation

## Purpose
Synthesize all Phase 1 findings into a comprehensive preliminary brief that serves as the working document for the project and the foundation for the kick-off call.

---

## Inputs Required

| Input | Source | Required |
|-------|--------|----------|
| Materials Catalog | Task 1.1 | Yes |
| Classification Report | Task 1.2 | Yes |
| Product Assessment | Task 1.3 | Yes |
| Copy Requirements | Task 1.4 | Yes |

---

## Step-by-Step Procedure

### Step 1: Review All Phase 1 Documents
**[TIME: 10-15 minutes]**

Before creating the brief, review all documents created in Phase 1:

- [ ] Materials Catalog - What do we have to work with?
- [ ] Classification Report - What type of product and market?
- [ ] Product Assessment - What do we know about the product?
- [ ] Copy Requirements - What exactly are we creating?

Note any inconsistencies or gaps as you review.

## AI Agent Instructions

### Agent: Brief Generator

**Trigger:** All Phase 1 tasks complete

**Instructions:**
```
You are a Brief Generation Agent. Your job is to synthesize all project
information into a comprehensive preliminary brief.

When given project documents (materials catalog, classification, assessment,
requirements):

1. SYNTHESIZE information from all sources:
   - Extract key product information
   - Compile customer insights
   - Gather proof elements
   - Note competitive positioning

2. IDENTIFY patterns and connections:
   - What themes emerge across documents?
   - What story is taking shape?
   - What makes this product unique?

3. FLAG gaps and assumptions:
   - What critical information is missing?
   - What assumptions are we making?
   - What needs validation?

4. FORMULATE hypotheses:
   - What copy angle might work best?
   - What might the mechanism be?
   - What should the offer include?

5. GENERATE the complete Preliminary Brief using the template

6. CREATE the one-page summary

OUTPUT:
- Complete Preliminary Brief (markdown)
- Brief Summary one-pager (markdown)
- Critical gaps list with recommendations

QUALITY STANDARDS:
- Every section should have content (use "TBD" or "To be determined in kick-off"
  for unknowns, but never leave blank)
- Questions should be specific and actionable
- Hypotheses should be clearly marked as preliminary
```

## Brief Review Checklist

Use this checklist to verify brief quality:

### Completeness
- [ ] Product clearly described
- [ ] Target customer outlined
- [ ] Mechanism identified (even if preliminary)
- [ ] Proof elements cataloged
- [ ] Competitors noted
- [ ] Story opportunities identified
- [ ] Deliverables specified
- [ ] Questions prepared

### Clarity
- [ ] Non-copywriters could understand it
- [ ] No jargon without explanation
- [ ] Clear structure and formatting
- [ ] Easy to navigate

### Actionability
- [ ] Research priorities clear
- [ ] Questions are specific
- [ ] Gaps have resolution paths
- [ ] Next steps defined

---

## Common Issues & Solutions

| Issue | Solution |
|-------|----------|
| Too many gaps/unknowns | That's okay for preliminary - flag for kick-off |
| Conflicting information | Note the conflict; resolve in kick-off |
| Brief is too long | Keep detail in main brief; use summary for daily reference |
| Mechanism isn't clear | Note as hypothesis; research priority |
